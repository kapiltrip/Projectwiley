Cell delay is the time taken for the cell output to switch logic  
from the time at which cell input switched logic in a way that  
causes the output to change. Cell delay is a function of the input  
transition time and the effective load at the output pin. The effec-  
tive output load is a combination of the net delays and the input  
capacitance at the subsequent cell input driven by this net. There-  
fore, the waveform delay and transition (slew) time are the two  
waveform parameters widely used in the non-linear delay model,  
NLDM, used to characterize logic gates to perform timing closure  
as explained in [8] and [1]. As the transistor feature size decreases,  
and got adopted as a Silicon Integration Initiative standard [11],  
and Synopsys introduced the Composite Current Source Model,  
CCS, [13]. Both ECSM and CCS depend on storing waveform infor-  
mation the library technology ﬁle to be able to model cell delays  
of more complex nature.  
In NLDM, Delay and transition time lookup tables are created  
during the cell characterization process to model the cell delay  
and output transition time. Delay and transition timetables specify  
the corresponding timing value for few selected values of input  
waveform transition time and effective capacitance. Interpolation  
and extrapolation techniques are used to extract the timing values  
for input transition and capacitive load values that do not exist in  
the tables. On the other hand, deep-learning techniques [4] can  
generalize based on its trained dataset. Trained feed-forward neu-  
ral network can better predict values beyond its trained dataset  
more accurately. Voltage and current waveforms are also used to  
obtain effective current source model and composite current  
source models respectively. Waveform compression techniques  
are needed to increase the waveform ﬁdelity without increasing  
the technology ﬁle size. General data compressors can be used like  
gzip, zlib, and bz2 techniques [3]. Ramalingam et al. [2] proposed  
ﬁxed waveform-basis principal component analysis. PCA, using  
the singular value decomposition, SVD, technique. Hatimi et al.  
[9] used an adaptive waveform-basis SVD to get better compres-

2. Cell-delay modeling background  
Cell characterization is the process of extracting important cell  
characteristics to build an accurate and efﬁcient cell model that  
represents its behavior. The approach of trading accuracy with run-  
time performance is usually taken to guarantee adequate cell  
model that executes fast in different design ﬂows. This paper  
focuses on voltage-based cell driver models; NLDM model and  
ECSM waveforms.  
2.1. NLDM cell driver model  
The most notable cell driver model is the NLDM as explained by  
Sharma [8]. The characterization process starts with connecting  
the cell output pin to an effective load capacitance and applying  
a ramp signal to the cell input. The ramp slew rate and the effective  
load capacitance are swept through deﬁned a range to measure the  
output slew rate and transition time. Lookup tables of output slew  
rate and transition time parameters at in given input slew rate and  
the effective load capacitance are stored in technology ﬁles. Usu-  
ally, two tables exist, one for the rising-edge and the second for  
the falling-edge. Static timing analysis tools make use of these  
technology tables to perform its timing analysis. NLDM ignores  
the maximum and minimum waveform values and all other wave-  
form ﬂuctuations as if focuses on transition time and delay. As the  
transistor feature size decreases, and the complexity of designs  
increases, timing analysis becomes more sensitive to signals  
over-shoot, under-shoot, and multiple crossings. Hence, more  
complex models, like ECSM, were introduced to increase cell model  
accuracy.

3. Deep learning Autoencoders background  
4.1.1. Generating NLDM data and ECSM waveforms  
Feed-forward neural network is used to encode large data into a  
smaller set of parameters representing the original data. The core  
idea of Autoencoder is to train a deep feed-forward neural network  
to produce an output that matches the input data to a certain  
degree of error [4]. Zhang et al., [12], show several new applica-  
tions to Autoencoders in convolutional neural networks, natural  
language processing, extreme learning machines and others. Wang  
et al., [16], concluded that Autoencoders can detect repetitive  
structure in addition to performing dimensionality reduction of  
trained data. The output layer must have the same size of the input  
layer. If the Autoencoder middle layer size is smaller than the num-  
ber of inputs as shown in Fig. 1, then the Autoencoder can perform  
dimensionality reduction of data which the main usage of Autoen-  
coders in this research. After training, the autoencoder is split at  
the middle layer into two models: an encoder model and a decoder  
model. The layers from the input layer to the middle layer repre-  
sent the encoder model. When the original data is presented as  
input to the encoder model, the output values of the middle layer  
neurons are the encoded, compressed data of the original data.  
Therefore, the number of neurons in the middle layer is considered  
the number of encoding parameters. The layers from the middle  
layer to the output layer represent the decoder model with the  
middle layer playing as the input layer to the decoder model. The  
compressed encoded data – the output of the middle layer, are pre-  
sented as input to the ﬁrst decoder model layer to be decoded to  
reproduce the original data within certain degree of error.  
MOS transistor model level 53 is used to perform transistor  
level simulation of NOR, NAND and INV cells. Fig. 2 shows the cell  
characterization setup; each cell is connected to a capacitive load  
and 1.2v VDD source. The slew rate of a ramp input is varied from  
10 ps to 2 ns, whereas the capacitive load is varied from 0.1fF to  
100fF.  
In each simulation run, the 1000-points voltage–time samples  
are measured at the output pin to store the waveforms needed  
for ECSM models. Uniform time 1000 sampled points per wave-  
form is selected to capture more complex waveform properties like  
overshoots, undershoots, rapid transitions and multiple crossings.  
Another sampling technique is devised to sample the most signif-  
icant N samples before crossing 95%VDD or 5%VDD for rising or  
falling time waveforms respectively as shown in Fig. 3. The number  
of samples N is varied between 50, 100 and 150 points. Sampling  
150-points gave better results than 50 and 100, therefore sampling  
results of 150-points are reported and compared with sampling  
uniform time 1000-points in this paper.  
Total of 71,400 rising-edge and another 71,400 falling-edge  
voltage waveforms were captured for ECSM models. During the  
same simulation run, delay and transition time values are recorded  
against the input waveform transition time and effective load  
capacitance to train the NLDM deep learning models.

4.4. Deep learning Waveform-Delay Model, DL-WFDM  
Deep learning waveform delay model, DL-WFDM, is proposed  
instead NLDM LUT or DL-NLDM models. In DL-WFDM, auto-  
encoded input/output waveforms parameters are proposed to  
model cell delays instead of the cell delay and the transition time  
used in the NLDM. Fig. 5 shows our proposed new cell delay train-  
ing methodology. First, each input waveform is encoded as well as  
its associated output waveform using the corresponding rising or  
falling edge Autoencoder encoder model. Since sigmoid activation  
function is used for the middle Autoencoder layer i.e., the encoder  
output layer, there is no need to normalize the input waveform  
parameters or the expected output waveform parameters.  
The encoded input waveform parameters and the associated  
effective capacitance values are applied as input data to train the  
deep learning network. The deep-learning network is trained on  
these encoded input waveform parameters in addition to the asso-  
ciated effective capacitance input value to produce the expected  
encoded output waveform parameters.  
Table 8 shows that the proposed DL-WFDM model results.  
Using 1000-points sampled encoded waveforms to train the new  
model resulted in up to 1.14% average percentage error, and up  
to 18.13% maximum percentage errors, for key waveform points  
compared with SPICE simulation. Using 150-points sampled  
encoded waveforms, DL-WFDM model gives up to 1.33% average  
percentage error, and up to 20.57% maximum percentage errors.  
Comparing the delay time errors in Tables 2, 3. 4 with the 50%  
VDD crossings in Tables 8 & 9 shows that DL-WFDM gives slightly  
more average percentage errors, but better maximum percentage  
The same experiment is repeated with 150-points sampled  
waveforms at variable time range. Table 7 shows that Autoen-  
coders of 2 parameters also give 1.79Â better compression ratio  
results than the nearest SVD model of rank 4, and less than  
0.45% average percentage error compared with SPICE simulation.  
Autoencoders of 2 parameters also gave between 40Â and 55Â bet-  
ter compression ratios than gzip and bz2 techniques at less degree  
of accuracy. This accuracy is achieved with ofﬂine training time of  
100 s for 100 epochs. The encoded 150-points sampled waveforms  
remain overall better than the encoded 1000-points sampled  
waveforms in average and maximum percentage errors. Fig. 4  
shows reconstructed falling and rising edge waveform example.  
Autoencoding either 1000 and 150 points waveforms with 4  
parameters gives smaller errors rates than the reported SVD rank  
16 and rank 4 error rates yet they are still greater than SVD rank  
32, and rank 8 error rates respectively. The smaller error rates with  
4 parameters encoding come at the cost of reduced compression  
ratios, but it remains a tradeoff option. However, encoding wave-  
forms with larger parameters doesn’t signiﬁcantly reduce the error  
rates.

5. Conclusions and future work  
Declaration of Competing Interest  
In this research paper, DL-NLDM model is proposed, designed,  
and trained on input and output waveform delay and transition  
time in addition to capacitive load values. Two cell-delay models  
are trained on rising and falling edge waveform transitions.  
Average delay percentage error values of less than 1.4% is  
achieved compared to SPICE simulation outperforming average  
percentage errors of the standard 7Â7 LUT size and outper-  
formed the non-standard 100Â100 LUT in maximum percentage  
errors. Waveform compression allows storing more complex  
waveforms data to raise the accuracy of the current source mod-  
els without increasing the technology ﬁle size or degrading the  
performance. In this research, deep learning non-linear Autoen-  
coders are used to compress voltage–time waveforms used in  
effective current source model. The performance of several deep  
learning Autoencoder models at different number of encoding  
parameters is evaluated against the SVD compression technique.  
Compression ratio of 104 at up to 0.66% average percentage  
error is achieved after encoding 1000-points sampled voltage–  
time waveforms. This compression ratios are 1.67Â better than  
the nearest rank SVD results and 39 to 45x better than gzip  
and bz2 compression techniques. Better compression ratios are  
also achieved at less accuracy ﬁgures with other Autoencoder  
models and number of parameters. Autoencoding 150-points  
sampled waveforms with 2 parameters gives 1.79Â better com-  
pression ratio than the nearest rank SVD compression with up to  
0.45% average percentage error compared with SPICE simulation.  
Those Autoencoders gave 40 to 55x better than gzip and bz2  
compression techniques. Autoencoders require large time to be  
trained, and larger compression and decompression time com-  
pared to the SVD algorithm, however training and compression  
are ofﬂine operations that can be accepted for the sake of better  
compression and accuracy. The decompression time can be  
enhanced with better hardware resources and smaller encoding  
models.

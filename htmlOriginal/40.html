Abstract—In this brief, we propose an effective adaptation of viability  
analysis in statistical static timing analysis. The adaption beneﬁts well  
from a dynamic programming implementation of the viability function.  
For a rapid identiﬁcation of statistical longest true paths, the technique  
makes use of a fast preprocessing step identifying the gates with a  
small probability of being viable in the circuit, and a number of simple  
optimization techniques. This makes the approach fast without lowering  
its accuracy. The efﬁcacy of the proposed statistical timing analysis  
is assessed using ISCAS benchmark circuits and carry skip adders.  
The results show that the proposed technique leads to, on average,  
18× higher speed compared to those of the state-of-the-art technique. This  
improvement is achieved at the cost of −1.7% precision lost compared  
to that of the Monte-Carlo method.

I. INTRODUCTION  
Static timing analysis (STA) is invoked to identify the critical path  
of circuit in a design cycle without any simulation. Only true critical  
paths, which can be sensitized by proper input vectors, have impact on  
the circuit clock frequency [1]. In dynamic path sensitization method,  
in contrast to the static one, temporal and transitional values on the  
input pins of each gate in the path are considered in the analysis  
[1], [2]. In the presence of the variations, criticality and trueness  
of paths may vary in different instances of the fabricated circuit [3].  
Therefore, in statistical STA (SSTA), delays and trueness probabilities  
of paths are modeled by statistical distributions [3]. In this analysis,  
instead of one critical path, a set of the most probable longest paths  
should be considered.  
A few works which used SSTA for identifying critical paths  
contemplate false (not true) paths in their proposed methods (see  
[2], [4]–[6]). In [4], the dependencies of gate delays to the gate input  
vectors were analyzed which were joined by identifying the impact  
of false paths expected to be critical under variations. The results  
of [5] for ISCAS’89 benchmark circuits indicated that a signiﬁcant  
portion of the longest paths were actually false. The algorithm of  
[6] was based on a branch and bound framework which pruned

The best-ﬁrst procedure, as a common search method for ﬁnding  
the longest path [7], maintains the partial true paths in a priority  
queue ordered by the potential full length, or the length of the longest  
extension of the partial path. This quantity (i.e., potential full length)  
is named the esperance (ESP) of the partial path. The search in the  
best-ﬁrst procedure is terminated when an output is reached. This is  
due to the fact that if such a longer path exists, it should be identiﬁed  
sooner in the best-ﬁrst search process. The best-ﬁrst search generally  
examines fewer paths in comparison to the depth-ﬁrst search [7].  
The most reasonable quantity that may be chosen for expressing  
the ESP (length of the longest extension) of a partial path is the  
required time (RT) of nodes. The RT is deﬁned as the maximum  
time needed for a transition to travel from the node to a primary  
output. If a node has a longer RT, there will be a higher probability  
of being critical for the paths passing through this node. The RTs,  
which are obtained by performing conventional block-based SSTA  
on the circuit, are pessimistically inaccurate. We, however, use it as  
a ﬁrst step to provide a set of candidate paths which are processed  
further to determine if they can be considered as critical viable paths.  
The viability of a path depends on all the adjoining paths having  
equal or longer lengths compared to this path length [1]. It is required  
that the viability function be computed for each side input to each  
encountered node on the (candidate) path. In the best-ﬁrst search  
method, before evaluating the candidate path, every side path, longer  
than the candidate path, has been traced. This feature of the method  
enables us to avoid using a time-consuming recursive approach for  
ﬁnding longer side viable paths. The best-ﬁrst search method may be  
implemented using the dynamic programming approach explained in  
detail in [1].

B. Proposed Viability Function Under Delay Variations  
As in the primary viability analysis, the RT of the node is used as  
ESP. The approach to reduce the required computations for analyzing  
the viability of a node is explained as follows. Since every viability  
quantity becomes probabilistic, at each terminating node of a partial  
path, we just determine if this node has large enough viability (can be  
viable). The exact probability value of the viability is not important  
here. The reason is that we wish to assess that if this partial path can  
terminate on an output node yielding a path with the longest delay  
distribution. If a node does not have the required conditions, it is  
omitted from further computations.  
To assess the viability of a candidate fan-out node g to extend  
a partial path, each side input k to g must satisfy one of the  
following conditions: First, k must have a sensitizing value (the  
simplest condition). Otherwise, at least one longer viable path must  
terminate on k. Since in the proposed viability function, the exact  
computations needed for obtaining the viability distribution of the  
node is avoided, the list of such paths for each side input k of g  
is held. Therefore, instead of maintaining the exact distributions of  
the viabilities, knowing the number of longer side paths is enough  
here. If the number of such paths is 0 for a side input, it is still  
not certain that the node cannot be sensitized. The reason is that the  
strategy in the dynamic implementation is to trace partial paths with  
higher ESPs. This means that there might be viable side paths having  
greater delay distributions with lower probabilities. These side paths  
have not been encountered until now due to the employed best-ﬁrst  
search strategy.  
To consider the effect of such paths, instead of tracing back all side  
paths toward the input nodes of circuit, we use a heuristic method  
where the arrival times (ATs) of nodes are considered as a metric.  
The AT of the node is deﬁned as the delay of the longest partial path  
from the primary inputs of the circuit into the node. For the node g, if  
the side input k has a sensitizing value or its list of terminating longer  
viable paths is not empty, k can sensitize the path passing through the  
online input f of g. Otherwise, the viability function compares (by  
calculating TP) the AT of k and the delay of the candidate partial path  
III. VIABILITY ANALYSIS CONSIDERING PROCESS VARIATION  
Under the process variation, gate delays have statistical behaviors  
being usually expressed by random variables. In this brief, to model  
gate delays we use the ﬁrst order canonical model of [8], and to  
properly include spatial correlation of process parameters the grid  
model of [9] is utilized. Here, to determine the relative greatness of  
the path lengths, one may use the tightness probability (TP). For two  
random variables X and Y, the TP of X is the probability of X > Y.  
For comparing two random variables, we use the TP formulas in [10],  
which is also used in [8], for computing the MIN/MAX operations.

C. Improvement in ESP Factor  
To improve the quality of the ESP factor, we propose a pruning  
method. The good selection of ESP reduces the number of paths to be  
evaluated. The pruning method also uses ATs in a preprocessing step  
before beginning the main procedure for ﬁnding the longest viable  
paths. For every node of a circuit, a counter is deﬁned for each of its  
inputs. If an input has a NCV, then the counters of other inputs are  
incremented, indicating that this input has the required condition to  
sensitize them. On the other hand, if it has a CV, its AT is compared  
to the ATs of each of the other inputs. If the AT of this input is greater  
than that of the other ones, this node has a high chance of acting as a  
late side input sensitizing the other inputs. Hence, the counters of the  
other inputs should be incremented. Then, during ﬁnding the RTs of  
the nodes, if an input counter of a circuit node has a value less than  
the number of node inputs minus one, it will indicate that at least  
one of its side inputs does not satisfy the viability conditions. Thus,  
it is pruned from the RT calculation. This way, the ESP directs the  
partial paths to the nodes with higher viability probabilities.

viable paths under the process variation. The function viability

FindKLongestTruePaths. The latter function is the implementation

longest paths. Here, the technique proposed in [2], which efﬁciently  
sensitized the longest viable paths in the presence of variation, has  
been adopted. Then, for each extracted vector, a simple forward SSTA  
analysis is performed from the input nodes of the circuit to ﬁnd  
the AT distributions for each node. Finding ATs is accompanied by

the pruning. Then, a simple backward SSTA from the output nodes

after taking the above steps. This function ﬁnds the K longest viable  
paths of the circuit in the presence of the delay variation and stores  
them in a related list descending. For several input vectors, this  
procedure is performed and at the end, the K top longest viable  
paths in the list are chosen as the ﬁnal result of the algorithm.

V. RESULTS AND DISCUSSION  
In this brief, six variational process parameters of length, width,  
and threshold voltage for nMOS and pMOS transistors with the  
maximum variation of (σ/μ) of 15% were considered. Sequential  
ISCAS benchmarks as well as a number of carry skip adder (CSKA)  
circuits were used as test circuits. For each circuit, a maximum  
number of 20 critical paths (K) were found. For extracting the good  
input vectors in the primary viability analysis, we used the SAT solver  
of [11]. We implemented our algorithm in C++ and ran it on a  
Linux machine with an Intel Core i7 CPU (4790 at 3.60 GHz) and  
8-GB RAM. Gate delays and sensitivities of the delay quantities to  
parameter variations were obtained for each gate input separately.  
Here, SPICE simulations were used with a SPICE library of the  
NanGate45 package [12]. Since our study was focused on assessing  
the efﬁcacy of the methods, all the results presented here are for runs  
under one input vector.

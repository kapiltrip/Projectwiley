Abstract. In this brief, we propose an effective adaptation of viability analysis in statistical static timing analysis. The adaption benefits well from a dynamic programming implementation of the viability function. For a rapid identification of statistical longest true paths, the technique makes use of a fast preprocessing step identifying the gates with a small probability of being viable in the circuit, and a number of simple optimization techniques. This makes the approach fast without lowering its accuracy. The efficacy of the proposed statistical timing analysis is assessed using ISCAS benchmark circuits and carry skip adders. The results show that the proposed technique leads to, on average, 18× higher speed compared to those of the state-of-the-art technique. This improvement is achieved at the cost of −1.7% precision lost compared to that of the Monte-Carlo method.

For a rapid identification of statistical longest true paths, the technique makes use of a fast preprocessing step identifying the gates with a small probability of being viable in the circuit, and a number of simple optimization techniques. II. VIABILITY

Static timing analysis (STA) is invoked to identify the critical path of circuit in a design cycle without any simulation. Only true critical paths, which can be sensitized by proper input vectors, have impact on the circuit clock frequency [1]. In dynamic path sensitization method, in contrast to the static one, temporal and transitional values on the input pins of each gate in the path are considered in the analysis [1], [2]. In the presence of the variations, criticality and trueness of paths may vary in different instances of the fabricated circuit [3]. Therefore, in statistical STA (SSTA), delays and trueness probabilities of paths are modeled by statistical distributions [3]. In this analysis, instead of one critical path, a set of the most probable longest paths should be considered. A few works which used SSTA for identifying critical paths contemplate false (not true) paths in their proposed methods (see [2], [4]–[6]). In [4], the dependencies of gate delays to the gate input vectors were analyzed which were joined by identifying the impact of false paths expected to be critical under variations. The results of [5] for ISCAS’89 benchmark circuits indicated that a significant portion of the longest paths were actually false. The algorithm of [6] was based on a branch and bound framework which pruned subpaths that were not true, or their criticality probabilities were low. The run times of these works were long due to unnecessary computational overheads. In addition, many of them were using satisfiability (SAT) solvers and static sensitization method which were imprecise. Also, they did not consider explicitly variations in checking the sensitization of gates. In [2], a viability analysis technique, introduced in [1], which considers statistical gate delay

One of the well-known dynamic sensitization methods is the viability analysis, where a viability function is utilized for checking the sensitization of each node in the path. The function considers the ability of other nodes in the circuit in sensitizing each node [1]. An input to a node lying on a given path is named online input while the other inputs of the node are named side inputs. The late side input is a side input which has a transition from noncontrolling value (NCV) to controlling value (CV) after the transition on the on-line input. Thus, it temporarily can sensitize the node [2]. To analyze whether a gate can propagate a transition from one of its input pins, for a given input vector v, the viability analysis discovers a number of paths which allow passing of transitions originated from the primary inputs of circuit to the gate inputs. These paths are called viable paths [1]. More specifically, [1] stated that a path P, consisting of the nodes { f0, . . . , fm}, is viable under an input vector v, iff, for each fi and each side input g ̸= fi−1 to fi, either 1) g is set to its sensitizing value by v or 2) g terminates a viable path under v with a delay longer or equal than the delay of the partial path { f0, . . . , fi−1}. The best-first procedure, as a common search method for finding the longest path [7], maintains the partial true paths in a priority queue ordered by the potential full length, or the length of the longest extension of the partial path. This quantity (i.e., potential full length) is named the esperance (ESP) of the partial path. The search in the best-first procedure is terminated when an output is reached. This is due to the fact that if such a longer path exists, it should be identified sooner in the best-first search process. The best-first search generally examines fewer paths in comparison to the depth-first search [7]. The most reasonable quantity that may be chosen for expressing the ESP (length of the longest extension) of a partial path is the required time (RT) of nodes. The RT is defined as the maximum time needed for a transition to travel from the node to a primary output. If a node has a longer RT, there will be a higher probability of being critical for the paths passing through this node. The RTs, which are obtained by performing conventional block-based SSTA on the circuit, are pessimistically inaccurate. We, however, use it as a first step to provide a set of candidate paths which are processed further to determine if they can be considered as critical viable paths. The viability of a path depends on all the adjoining paths having equal or longer lengths compared to this path length [1]. It is required that the viability function be computed for each side input to each encountered node on the (candidate) path. In the best-first search method, before evaluating the candidate path, every side path, longer than the candidate path, has been traced. This feature of the method enables us to avoid using a time-consuming recursive approach for finding longer side viable paths. The best-first search method may be implemented using the dynamic programming approach explained in detail in [1].

III. VIABILITY ANALYSIS CONSIDERING PROCESS VARIATION

Under the process variation, gate delays have statistical behaviors being usually expressed by random variables. In this brief, to model gate delays we use the first order canonical model of [8], and to properly include spatial correlation of process parameters the grid model of [9] is utilized. Here, to determine the relative greatness of the path lengths, one may use the tightness probability (TP). For two random variables X and Y, the TP of X is the probability of X > Y. For comparing two random variables, we use the TP formulas in [10], which is also used in [8], for computing the MIN/MAX operations. Jung and Kim [2] proposed an approach (which is used in this brief) for converting Boolean formulas (1)–(3) into a form which considers the statistical nature of the quantities under process variation. They utilized the theorem which states that logical AND/OR operations and statistical MIN/MAX operations are equivalent when it is needed to know that the probability of a random variable being larger than zero. This equivalence results in (4)–(6) are given in Table I. The viability (probability of a path to be viable) of the path P under an input vector γ is defined as Prob[φP(γ ) > 0] [2]. Here, φP(γ ) is the viability distribution of P under γ .

A. Simplification in Viability Formulas Under a Test Vector

As in the primary viability analysis, the RT of the node is used as ESP. The approach to reduce the required computations for analyzing the viability of a node is explained as follows. Since every viability quantity becomes probabilistic, at each terminating node of a partial path, we just determine if this node has large enough viability (can be viable). The exact probability value of the viability is not important here. The reason is that we wish to assess that if this partial path can terminate on an output node yielding a path with the longest delay distribution. If a node does not have the required conditions, it is omitted from further computations. To assess the viability of a candidate fan-out node g to extend a partial path, each side input k to g must satisfy one of the following conditions: First, k must have a sensitizing value (the simplest condition). Otherwise, at least one longer viable path must terminate on k. Since in the proposed viability function, the exact computations needed for obtaining the viability distribution of the node is avoided, the list of such paths for each side input k of g is held. Therefore, instead of maintaining the exact distributions of the viabilities, knowing the number of longer side paths is enough here. If the number of such paths is 0 for a side input, it is still not certain that the node cannot be sensitized. The reason is that the strategy in the dynamic implementation is to trace partial paths with higher ESPs. This means that there might be viable side paths having greater delay distributions with lower probabilities. These side paths have not been encountered until now due to the employed best-first search strategy. To consider the effect of such paths, instead of tracing back all side paths toward the input nodes of circuit, we use a heuristic method where the arrival times (ATs) of nodes are considered as a metric. The AT of the node is defined as the delay of the longest partial path from the primary inputs of the circuit into the node. For the node g, if the side input k has a sensitizing value or its list of terminating longer viable paths is not empty, k can sensitize the path passing through the online input f of g. Otherwise, the viability function compares (by calculating TP) the AT of k and the delay of the candidate partial path ending onto f . If the AT is greater than the delay of the partial path,

To improve the quality of the ESP factor, we propose a pruning method. The good selection of ESP reduces the number of paths to be evaluated. The pruning method also uses ATs in a preprocessing step before beginning the main procedure for finding the longest viable paths. For every node of a circuit, a counter is defined for each of its inputs. If an input has a NCV, then the counters of other inputs are incremented, indicating that this input has the required condition to sensitize them. On the other hand, if it has a CV, its AT is compared to the ATs of each of the other inputs. If the AT of this input is greater than that of the other ones, this node has a high chance of acting as a late side input sensitizing the other inputs. Hence, the counters of the other inputs should be incremented. Then, during finding the RTs of the nodes, if an input counter of a circuit node has a value less than the number of node inputs minus one, it will indicate that at least one of its side inputs does not satisfy the viability conditions. Thus, it is pruned from the RT calculation. This way, the ESP directs the partial paths to the nodes with higher viability probabilities. The pruning method may be explained better using the fourinput AND gate g of Fig. 1(a). The initial values of all the counters (cnt1, . . . , cnt4) are considered to be 0. Since the inputs 3 and 4 have the sensitizing value, the counters become (2, 2, 1, 1) [input 3 →(1, 1, 0, 1); input 4 →(1 + 1, 1 + 1, 0 + 1, 1 + 0)]. Since the input 1 has the CV and its AT has the greatest distribution using the TP among the distributions of other ATs, it could be act as a late side input for the other inputs. Thus, the counters are updated to (2, 3, 2, 2). Input 2 also has the CV and its AT is greater than those for inputs 3 and 4. Thus, at the end, the counter values become (2, 3, 3, 3). Since cnt2, cnt3, and cnt4 have values equal to the number of the inputs of g minus one, inputs 2 through 4 are not pruned, while input 1 is pruned. Note that if all the inputs of the gate g have the sensitizing value, no node will be pruned. On the other hand, if all the inputs have the CV, only input 4, which has the smallest AT, can be sensitized.

E. Overall Algorithm

Algorithm 1 shows the overall procedure for finding the K longest viable paths under the process variation. The function viability function is explained in Section ýIV-B and is used in the function FindKLongestTruePaths. The latter function is the implementation of the dynamic programming procedure for finding the K longest viable paths (which is path-based) as explained in [1]. It considers the delay variation. The overall procedure begins with extracting a set of useful input vectors providing the conditions for sensitizing the top longest paths. Here, the technique proposed in [2], which efficiently sensitized the longest viable paths in the presence of variation, has been adopted. Then, for each extracted vector, a simple forward SSTA analysis is performed from the input nodes of the circuit to find the AT distributions for each node. Finding ATs is accompanied by finding the static values of the circuit nodes along with performing the pruning. Then, a simple backward SSTA from the output nodes of the circuit is executed to find the RT distributions of all the nodes obtaining their ESPs. The function FindKLongestTruePaths is called after taking the above steps. This function finds the K longest viable paths of the circuit in the presence of the delay variation and stores them in a related list descending. For several input vectors, this procedure is performed and at the end, the K top longest viable paths in the list are chosen as the final result of the algorithm.
